Technical Specification

UnlockDB is a high-performance, in-memory NoSQL database server leveraging the Jint JavaScript Engine for programmable query execution. This documentation provides a complete reference for integration, training AI models, and mastering the JavaScript query interface.

Architecture

The system operates as a stateful REST API where query sessions are independent. Each request creates a fresh Jint engine scope to ensure isolation and security.

  • Engine: Jint 4.4.2 (.NET Integration)
  • Storage: ConcurrentDictionary with disk persistence (JSON)
  • Concurrency: Thread-safe CRUD via atomic memory operations

Curl Examples

You can interact with the UnlockDB API directly using curl or any HTTP client. All queries should be sent as raw text in the request body.

1. List Collections

curl -X GET http://localhost:5000/collections \
  -H "Authorization: Basic dW5sb2NrZXI6dW5sb2NrZXI="

2. Simple Query (Find All)

curl -X POST http://localhost:5000/query \
  -H "Authorization: Basic dW5sb2NrZXI6dW5sb2NrZXI=" \
  -d "db.users.findall().ToList()"

3. Filtered Query

curl -X POST http://localhost:5000/query \
  -H "Authorization: Basic dW5sb2NrZXI6dW5sb2NrZXI=" \
  -d "db.users.findall(x => x.age > 20).ToList()"

4. Insert Record

The 'db' Object

Queries are executed against the global db object. You can access any collection dynamically using property notation.

// Accessing collections
const users = db.users; 
const products = db.products;

Finding Data

findall(predicate) Returns: ResultSet

Retrieves records matching the predicate. If omitted, returns all records.

// Get all
db.users.findall().ToList();

// Filtered
db.users.findall(x => x.age > 25).ToList();
find(predicate) Returns: Document

Returns the first record matching the criteria (Lazy evaluation).

const user = db.users.find(x => x.email == "admin@db.com");

Inserting Data

Every document automatically receives a unique _id (GUID) upon insertion.

db.users.insert({
    username: "dev_user",
    tags: ["nosql", "fast"],
    metadata: { lastLogin: new Date() }
});

Updating Data

Updates can be performed directly on a collection or on a ResultSet.

// Collection-level update (Bulk)
db.users.update(x => x.status == "pending", { status: "active" });

// ResultSet-level update
db.users.findall(x => x.points > 100).update({ level: "pro" });

Deleting Data

// Bulk delete via query result
db.users.findall(x => x.isDeleted == true).delete();

Aggregation & Joins

UnlockDB supports foreign-key style joins between multiple collections based on ID matching.

// Join users and their orders
const reports = db.join(db.users, db.orders).findall().ToList();

Projection (Select)

Use select() to transform results or extract specific fields.

// Map to new schema
const names = db.users.findall().select(x => ({ 
    fullName: x.firstName + " " + x.lastName,
    id: x._id 
}));

Limiting (Take)

Limit the number of results for paginated or preview views.

// Get top 5 newest records
db.logs.findall().take(5).ToList();

Custom JS Functions

You can use standard JavaScript logic (functions, loops, variables) to process data. The last evaluated expression or a returned value determines what appears in the results grid.

// Using a functional approach
(function() {
    const data = db.users.findall().ToList();
    const processed = data.map(u => {
        return {
            id: u._id,
            tag: u.age > 18 ? "Adult" : "Minor",
            raw: u
        };
    });
    return processed;
})();
Tip: Always use ToList() when you need to iterate over collection data in standard JavaScript arrays.

Utility Functions

UnlockDB includes a suite of built-in helper functions to simplify data manipulation and security tasks within your scripts.

Cryptography & Security

  • md5(string): Returns MD5 hash.
  • sha256(string): Returns SHA256 hash.
  • encrypt(text, salt) / decrypt(text, salt): Simple encryption helpers.
  • toBase64(string) / fromBase64(string): Text encoding.

Data Utilities

  • randomString(length): Generates random alphanumeric strings (great for tokens).
  • randomNumber(min, max): Generates random integers.
  • randomDecimal(minStr, maxStr): Generates random decimals.
  • split(text, separator): Splits string into array.
  • toString(obj): Safe string conversion.
Example: Secure User Creation 
db.users.insert({
    username: "new_user",
    password: sha256("mySecretPass"), 
    apiToken: randomString(64),
    createdAt: toString(new Date())
});

Index Management

Optimizing queries with indexes. Supported types: ASC (Ascending) and DESC (Descending).

// String-based creation
db.users.index("email", "ASC");

// Lambda-based creation
db.products.index(x => x.price, "DESC");
AI Training Note: UnlockDB uses "Text Indexing" for strings (word-based + n-grams) and "Sorted Indexing" for numeric values.

Lazy Evaluation

The engine defers data realization until ToList() or first() is called. This minimizes memory overhead during complex logical filtering.
